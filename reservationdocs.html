GetPreReservationPrice

Used to return the price of a prospective reservations rates, taxes and fees. The response will always include the Company's currency format as "distribution_channel_currency" field.

Name Data Type Description Required?
token_key String (255) The expiring token key credential provided to you by Streamline Yes
token_secret String (255) The expiring token secret credential provided to you by Streamline Yes
unit_id Number The unique ID for the property you are making a reservation or getting information from. Yes
startdate String (255) The checkin date for the reservation you are getting pricing for. Yes
enddate String (255) The checkout date for the reservation you are getting pricing for. Yes
occupants Number The number of adult guests staying at the property. Yes
occupants_small Number The number of child guests staying at the property. No
pets Number Specifies the number of pets that belong to a reservation. No
apply_reward_points Number Decides whether or not reward points will be applied toward the price of the reservation. No
coupon_code String (120) The coupon_code is factored into reservation pricing and rates. No
madetype_id Number The madetype ID for the reservation. This is handled naturally by the API and assumes that a reservation coming from this API will be an NET (internet) (9) reservation. If you need to use a different parameter, the values are: 1 = INET (Internet, deprecated, use NET instead) 2 = POST (Post-Deduct TA Reservation) 3 = FDR (Frontdesk Reservation) 4 = ADM (Admin Reservation) 5 = OWN (Owner Reservation) 6 = PRE (Pre-Deduct Reservation) 7 = WSR (Wholesale Reservation) 8 = BCR (Business Card Reservation, deprecated) 9 = NET (XML API Reservation) 10 = OSC (OSCARS Reservation, deprecated) 11 = PDWTA (Post-Deduct Wholesale TA Reservation) 12 = INNLINK (InnLink Reservation, deprecated) 13 = LAPI (Lead from API Reservation, deprecated) 14 = £MCW$ (Multi-currency WP Plugin Booking, deprecated) 15 = REP (Repeat, deprecated). No
type_id Number The type (reservation type, like Standard, Owner, Airbnb-NI, etc) to return pricing and room rate for. Default is 2 if not passed (2 = Standard). You can get the type_id's for a company by using the API GetReservationTypes. No
distributor_code String (255) The code tied to a particular distribution channel. Distribution channels are tied to a reservation type. If you are passing distributor_code, you do not need to pass made_type or make_type. No
pricing_model String Defines the pricing model to be used for the reservation. 1 is daily, 2 is weekly, 3 is monthly. Sometimes users will have all three and will want to use a specific rate. No
optional_fee_XXXX Boolean Activates an optional fee when {{XXXX}} is replaced with the fee ID, and the parameter is passed with a value of 1. For example, if the optional fee ID was 123, you would pass 'optional_fee_123: 1,' to activate the fee and have it impact pricing for a reservation. Used in websites to allow a guest to opt in or out of a fee using GetPreReservationPrice and allows an existing reservation to add or take away these fees by using GetReservationPrice. Use it with MakeReservation to actually add the fee onto the reservation. No
show_package_addons Null or Empty Outputs amenity packages if they are selected for the property and breaks them down by every amenity inside the package. No
optional_default_enabled Null or Empty Sets all Optional, Enabled By Default fees to active in response and also adds their amounts to the grand total of the reservation. Use this if you will be giving guests the ability to unselect optional fees and addons they don't want. No
return_payments Boolean Must be passed to return expected charges. No
payment_type_id Number The payment type that will be used to pay for the reservation. Important for users using charge type logic on fees, most commonly for a credit card surcharge. The list of payment types for a company can be gotten by using the GetPaymentTypes API method. No
include_coupon_information Boolean This includes the name and id of the autoapply coupon that impacts the total value of the reservation. No
separate_taxes Boolean Optional parameter, returns taxes_details_value and required_fees_value, allowing to see the totaled values for each type. No
guest_country_insurance_validation String (255) The shortcode for the country that the guest is from. Certain insurance is only applicable to guests from certain locations. If you pass this and the country is not in the list, then insurance won't be returned as one of your optional fees. If you don't pass the param, no validation will be used. No
guest_state_insurance_validation String (255) The shortcode for the state that the guest is from. Certain insurance is only applicable to guests from certain locations. If you pass this and the state is not in the list, then insurance won't be returned as one of your optional fees. If you don't pass the param, no validation will be used. No
return_new_pricing_model_and_rate_type Boolean If true (values: yes, 1), returns in response information about: New Pricing Model Id and New Rate Type Id assigned to recent calculations. Fields: new_pricing_model, new_rate_type_id. No
rate_type_id Number Applies the desired Rate Type ID that exists in PM's Streamline system No
adjust_rate Object Includes information about date and new price to change date string format: "mm/dd/yyyy" rate numeric JSON Example: { "0": { "date": "02/22/2022", "rate": 70 }, "1": { "date": "02/23/2022", "rate": 75 } } XML Example: 02/22/2022 70 02/23/2022 75 No
guest_deposits_show_all Boolean Will force guest deposits object to show even if it's not required today. No
show_due_today Boolean If PM has all Required CC Charge variables disabled, this parameter can be passed to allow to show amount for "due_today" response field. No
