MakeReservation

Used to make a reservation (General). Although several fields are marked as 'not required', a Streamline user has the ability to make website reservations require many optional parameters including email, cellphone, full address, etc. It's best practice to speak to the company you are working with to decide what all required fields will be before you begin testing.

payment_type_id Number ID for a payment type being posted with MakeReservation or AddReservationPayment. Currently there is no API to return payment types for a specific company. Global payment types are 1 = Charge CC, 4 = Charge CASH, 7 = Charge Owner, 11 = Wholesale Payment, 33 = E-Check/ACH. No

---   amenity_id Number The ID for the amenity addon you're adding to the reservation. No

optional_fee_XXXX String Activates an optional fee when {{XXXX}} is replaced with the fee ID, and the parameter is passed with a value of 1. For example, if the optional fee ID was 123, you would pass 'optional_fee_123: 1,' to activate the fee and have it impact pricing for a reservation. Used in websites to allow a guest to opt in or out of a fee using GetPreReservationPrice and allows an existing reservation to add or take away these fees by using GetReservationPrice. Use it with MakeReservation to actually add the fee onto the reservation. No

get_reservation_info String get reservation info (any value = true) No
language_country_id Number language country id (f.e.: 139 - spanish, 227 - english) No
corporation_id Number Is used for Corporate Reservations No

Recommended to not use this parameter, as the reservation will attempt to charge grand total if passed without this. If the company has autocharging or scheduled payments on, then reservations will charge the outstanding amount after this time has passed. No
credit_card_type_id Number 1 = Visa 2 = MasterCard 3 = AmericanExpress 4 = Discover No

credit_card_expiration_month Number Credit Card Expiration Month No
credit_card_cid Number Credit Card CID No

work_phone String Optional Work Phone Number No
hear_about_new String Optional parameter sets the source of the reservation. Important for property managers to know where reservations are coming from. Heard About source record will be created if one does not exist. (Main Menu -> Tools: Hear About Manager) No
hear_about_id Number Can be used over "hear_about_new". Accepts the ID for the existing Heard About source found in the PM system. No
phone String Phone Number No
credit_card_charge_required Boolean This forces the reservation to charge the credit card. No
travelagent_id Number The processor ID for the Travel Agent you are attaching to the reservation. You can get the processor ID's by using GetProcessors. No
rate_type_id Number Applies the desired Rate Type ID when creating a Reservation. Rate Type IDs can be found in Streamline Admin (Main Menu -> Tools -> Manage Rate Types) No
pricing_model String Defines the pricing model to be used for the reservation. 1 is daily, 2 is weekly, 3 is monthly. Sometimes users will have all three and will want to use a specific rate. No
coupon_code String (255) The coupon code for the coupon you want to apply to the reservation. If coupon_code is passed as empty string, no coupon will be applied. No
coupon_id Number coupon_id can be used in place of coupon_code No
referrer_url String The value set in parameter referrer_url will populate field Referrer field in Reservation Folio. No
payment_comments String (25550) Information about how the reservation was or will be paid for, if relevant. No
